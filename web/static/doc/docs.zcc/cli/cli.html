Chapter 2. Using the Control Center command-line interface

The Control Center command-line interface (CLI) and the Control Center web interface both use the same application programming interface. Any action performed in the CLI can be performed in the web interface.

To use the CLI, you need a Linux shell account on hosts in Control Center pools, and the account needs to be a member of the docker group.

Currently, CLI commands that are invoked on hosts other than the Control Center master host need to specify the master host and port manually. For example, to list running services, enter the following command on the master host: 

serviced service list

To avoid an error when invoking the same command on a host other than the master host, add the endpoint flag. 

serviced --endpoint Master-Host:4979 service list

Replace Master-Host with the hostname or IP address of the Control Center master host.

Note: To commit a container, a serviced CLI session must be run on the Control Center master host.
